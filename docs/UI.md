# SmartDoc UI 设计说明

## 1. 设计原则

- **清晰**：当前文件夹、目标格式、已选文件数一目了然。
- **防误操作**：转换前确认、转换中禁用按钮；失败不删原文件。
- **响应式**：单列布局，最大宽度约 4xl，适配桌面与平板。

## 2. 布局结构

```
+------------------------------------------+
| SmartDoc            [支持 PDF/EPUB/…]    |
+------------------------------------------+
| [选择文件夹]  当前: 未选择 / 文件夹名     |
| 目标格式: [下拉 PDF|EPUB|TXT|MOBI|AZW3]  |
| [ ] 使用大模型增强  模型: [下拉]        |
| [开始转换]  (N 个文件)                  |
+------------------------------------------+
| 文件列表                                 |
| [ ] 书名.pdf      PDF    等待           |
| [ ] 另一本.epub    EPUB   等待           |
| [全选当前类型] [仅 PDF] [仅 EPUB] …     |
+------------------------------------------+
| 转换状态（可与上合并）                   |
| 书名.pdf → 书名.epub  完成               |
+------------------------------------------+
```

## 3. 状态与样式

- **未选文件夹**：文件列表占位文案「请先选择文件夹」。
- **已选文件夹、未选文件**：「开始转换」禁用，提示「请至少勾选一个文件」。
- **已选文件、目标格式与源相同**：「开始转换」禁用，提示「请选择与源格式不同的目标格式」。
- **转换中**：对应行显示「转换中…」；「开始转换」「选择文件夹」禁用。
- **成功**：行尾显示「已替换」或绿色勾。
- **失败**：行尾显示「失败」红色，不删原文件。

## 4. 目标格式下拉逻辑

- 若选中文件均为同一扩展名（如全是 .pdf），则目标格式下拉**排除**该格式（不显示 PDF）。
- 若选中文件混合多类，目标格式下拉可显示全部，但转换时按「源扩展名 → 目标格式」逐文件处理；或限制为「仅允许选同类型文件再选目标格式」（产品可选）。

## 5. 技术实现要点

- 使用 Tailwind CSS 保持与现有风格一致（indigo 主色、圆角、阴影）。
- 文件列表使用可滚动区域（max-height），避免页面过长。
- 确认弹窗：原生 `confirm()` 或简单 modal（「将转换 N 个文件并删除原文件，是否继续？」）。
